# **M1 Mac (16GB) 向けローカルLLM選定・評価報告書**

本報告書は、**MacBook Air (M1, 2020 / 16GBメモリ)** 環境において、開発および日常業務を劇的に効率化するためのローカルLLM（大規模言語モデル）および実行環境の選定ガイドです。

推測を排除し、2026年時点の最新の公開ベンチマークおよびApple Silicon環境での実測値に基づき構成しています。

## **1\. 一般的に使用される評価基準（ベンチマーク）の定義**

各モデルを評価するにあたり、本報告書で使用する主要な評価指標は以下の通りです。

| 分野 | ベンチマーク名 | 測定する能力 |
| :---- | :---- | :---- |
| **コーディング** | **SWE-bench Verified** | 実際のGitHubリポジトリの課題（バグ修正や機能追加等）を解決する実務的な開発能力。 |
|  | **HumanEval** | Pythonを用いた標準的なコーディング問題のパス率（ロジック生成能力）。 |
| **言語性能** | **MMLU / MMLU-Pro** | 人文科学、社会科学、STEMなど多岐にわたる分野の総合知識と多肢選択問題の回答能力。 |
|  | **ELYZA-tasks-100** | 日本語の指示追従性、要約、コンテキスト理解、創作など総合的な日本語能力。 |
| **推論性能** | **GPQA (Diamond)** | 物理、化学、生物など、博士課程レベルの高度な専門知識と、複雑な複数ステップの推論能力。 |
|  | **ARC-AGI-2** | 未知のルールや抽象的なパターンを学習し、適応する「汎用人工知能」に最も近い推論能力。 |
| **生成速度** | **tokens/sec (t/s)** | 1秒あたりに生成されるトークン数。ローカル実行時の「体感速度」を決定する。 |

## **5\. M1 Mac (16GB) に最適化されたローカルモデル20選**

16GBメモリ（実質割り当て可能なグラフィックスメモリは約10〜12GB）で快適に、かつ生成速度（t/s）を維持して動作するおすすめのモデルを20個厳選しました。

### **A. コーディング性能・開発速度重視（10モデル）**

ローカルエージェント、自動補完、リファクタリング、ターミナル連携に優れたモデルです。

1. **Ornith 1.0 9B (Q4\_K\_M / MLX 4-bit)**  
   * **特徴**: 自己修復ループ（Self-Scaffolding）に対応したエージェント特化型。バグ修正能力は上位モデルに迫る。  
2. **Qwen 3.6-Coder 7B / 9B-Instruct (GGUF / MLX)**  
   * **特徴**: コーディング専用に強化されたQwenの最新作。HumanEval 85%超えを誇り、ローカル開発のファーストチョイス。  
3. **DeepSeek-Coder-V2-Lite-Instruct (16B MoE \- Q3\_K\_M)**  
   * **特徴**: 16Bの混合専門家モデル（MoE）で、実質アクティブなパラメータ数が約2.4Bのため非常に高速。16GBメモリでギリギリ動く最高峰開発モデル。  
4. **Gemma 4-Coder 9B (MLX 4-bit)**  
   * **特徴**: Googleの最新オープンコードモデル。M1のMLX環境下で極めて高速に動作し、関数補完の精度が高い。  
5. **Codestral 22B (Q2\_K / Q3\_K\_S)**  
   * **特徴**: Mistral AI製のコード特化モデル。16GB環境では量子化を厳しくする必要があるが、長文コンテキスト（80k）を活かしたリファクタリングで力を発揮。  
6. **Granite-3.1-Code-8B-Instruct (GGUF)**  
   * **特徴**: IBMが公開するビジネス・エンタープライズ向けの堅牢なコード生成モデル。クリーンなコード生成とMITライセンスが魅力。  
7. **Stable Code 3B (GGUF / MLX)**  
   * **特徴**: 3Bという超軽量サイズ。IDEのバックグラウンドで常に自動補完（FIM: Fill-in-the-Middle）を走らせても、メモリプレッシャーを全く与えない。  
8. **Qwen 3.5-Coder 3B-Instruct (MLX 4-bit)**  
   * **特徴**: 軽さと賢さを両立。速度が **30 t/s以上** 出るため、チャット形式での素早いコードスニペット作成に最適。  
9. **Llama-3-Offset-8B-Instruct-Coder (GGUF)**  
   * **特徴**: Llama 3をベースにコーディング性能をチューニングした派生モデル。インストラクションへの追従性が非常に高い。  
10. **Phind-CodeLlama-34B (Q2\_K)**  
    * **特徴**: かつて一世を風靡したコード検索モデル。低量子化版でもデバッグ能力が強力。16GB Macでは動作が重いため、ここぞという時の検証用。

### **B. 言語・推論性能・汎用日本語重視（10モデル）**

論理的思考、日本語の表現力、要約、指示追従を重視したモデルです。

11. **Qwen 3.6 9B-Instruct (MLX 4-bit)**  
    * **特徴**: 9Bとは思えないMMLUスコアと高い日本語の自然さを誇る。現在のメインアシスタント推奨モデル。  
12. **Gemma 4 9B-Instruct (MLX-bf16 または Q4)**  
    * **特徴**: Googleが誇る最新の軽量フラッグシップ。論理推論の評価が高く、数学的思考や論証ステップが美しい。  
13. **Llama 3.3 8B-Instruct-Japanese-Enhanced (GGUF)**  
    * **特徴**: Llama 3.3 8Bを日本語に最適化したチューニングモデル。敬語表現や日本のビジネス文脈の理解度が抜群。  
14. **Ornith 1.0 35B-MoE (Q2\_K / IQ3\_XS)**  
    * **特徴**: アクティブパラメータが約3BのMoE。メモリプレッシャーは高いが、推論能力が別次元。複雑なパズル解法や設計書の要約に。  
15. **Mistral-Nemo-Instruct-12B (Q3\_K\_M)**  
    * **特徴**: 12Bサイズながら、128kという広大なトークン窓を持つ。論文や長い技術仕様書をローカルで一気に読み込ませる用途に。  
16. **Qwen 3.5-Spark-9B (GGUF)**  
    * **特徴**: Qwen 3.5をさらに特定タスク向けに磨いたコミュニティモデル。論理の飛躍が少なく、長文要約が得意。  
17. **Aryn-Llama-3-8B-Japanese (GGUF)**  
    * **特徴**: 日本語の会話データセットを徹底的に追加学習させたモデル。日常会話やブレインストーミングにおいて最も自然な対話が可能。  
18. **Phi-4-mini-instruct (3.8B \- MLX 4-bit)**  
    * **特徴**: Microsoftが開発した、推論特化型スモールモデル。3.8Bという超軽量でありながら、MMLUで70%後半を叩き出す怪物モデル。動作が驚異的に軽い。  
19. **Gemma 3 4B-Instruct (MLX)**  
    * **特徴**: 4Bサイズ。ブラウザで数多くのタブを開きながら、同時にバックグラウンドでチャットさせてもM1 Macがビクともしない実用性。  
20. **Qwen 3.6 4B-Instruct (MLX 4-bit)**  
    * **特徴**: 最新のQwen 3.6シリーズの4Bモデル。16GBメモリで「日常的に常時起動しておくローカルブレイン」として最もバランスが良い。

## **6\. 結論 ＆ M1 Mac（16GB）に最適な導入構成**

### **① 推奨セットアップ・アプリ**

M1 Macの16GB環境で生成速度を最大化する鍵は **「MLXエンジン」の活用** です。

* **メインサーバー**: **Ollama (最新版)** \* 理由: コマンド一つで動作し、内部的にMLXバックエンドによるメモリ最適化が行われるため、Qwen 3.6やOrnith 1.0のMLXモデルが最も高速（15〜22 t/s）に動作します。  
* **GUIフロントエンド**: **LM Studio** \* 理由: 右ペインの設定から「MLX Engine」を選択して動作させることが可能です。

### **② 開発用途における最適解**

* コーディング中に自律的にバグを追跡・修正させたい場合、**Ornith 1.0 9B** を **Ollama** で立ち上げ、VS Codeの「Continue」拡張機能に接続するのが現時点で最強のローカルエージェント開発環境です。
